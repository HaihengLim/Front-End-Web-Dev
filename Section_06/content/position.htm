Z-index / stacking

When elements overlap, `z-index` controls stacking order. Only works on positioned elements (non-static).

z: 0
z: 1
z: 2
Tip: to make an ancestor the reference for `position: absolute`, give it any non-`static` position (commonly `position: relative`).

Quick cheat

/* Common position values */ .static { position: static; } /* default — normal flow */ .relative { position: relative; top:10px; left:10px; } /* offset, keeps original space */ .absolute { position: absolute; top:10px; right:10px; } /* removed from flow, positioned inside nearest positioned ancestor */ .fixed { position: fixed; top:10px; right:10px; } /* viewport-locked */ .sticky { position: sticky; top:0; } /* sticks when scrolled to offset */ /* z-index */ .positioned { position: relative; z-index: 10; }

Short notes:

  • Absolute elements do not take up space in the normal flow.
  • Relative only moves the element visually — layout space stays the same.
  • Fixed elements are viewport-relative; often used for toolbars/toasts.
  • Sticky is great for headers that pin while scrolling inside a container.
